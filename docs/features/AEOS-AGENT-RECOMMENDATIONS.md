# AEOS Agent Recommendations in Workspace

**Sprint:** MVP-AGENTS-3
**Status:** SHIPPED

---

## Summary

MVP-AGENTS-3 integrates the deterministic agent planner into every workspace
surface. The planner's risk analysis and recommended actions are now embedded
directly in:

1. **`aeos ui project-workspace`** — "Agent Recommendations" section in the HTML
2. **`aeos ui evidence-pack`** — `agent-plan.md` in every evidence pack
3. **`aeos workspace demo`** — `agent-plan.md` generated per project automatically

No LLM. No network. No .env. No registry mutation. Read-only. Applied: false.

---

## What changed

### 1. `src/aeos/agent/planner.py`

Added `generate_project_entry()` — a public function that analyses a single
project without needing a full registry. Takes `name`, `memory_dir`,
`project_type`, `evidence_dir`, and `workspace_dir` directly.

`_analyse_project()` is now a one-line wrapper that delegates to it, removing
the code duplication from before.

```python
from aeos.agent.planner import generate_project_entry

entry = generate_project_entry(name="my-project", memory_dir=Path("/path/to/memory"))
# entry.status → "OK" | "WARNING" | "ERROR"
# entry.risks  → list[str]
# entry.actions → list[str]
```

### 2. `src/aeos/ui/workspace.py`

- `WorkspaceData` has a new optional field: `agent_plan_entry: ProjectPlanEntry | None`
- `load_workspace_data()` calls `generate_project_entry()` and stores the result
- `render_workspace()` renders an "Agent Recommendations" section showing:
  - Planner mode label
  - Project status badge (OK / WARNING / ERROR)
  - Risks / Blockers list
  - Numbered recommended actions
  - `read_only: true · applied: false · human validation required` footer

### 3. `src/aeos/ui/evidence_pack.py`

- `_PACK_FILES` now includes `"agent-plan.md"`
- New `render_agent_plan_md(ws: WorkspaceData) -> str` — builds a full `AgentPlan`
  from the stored `agent_plan_entry` and delegates to `render_plan_markdown()`
- `generate_evidence_pack()` writes `agent-plan.md` alongside all other pack files
- `render_index()` lists `agent-plan.md` in the evidence pack index

### 4. `src/aeos/workspace/demo.py`

No changes needed — the demo calls `generate_evidence_pack()`, which now
automatically includes `agent-plan.md` for every project.
